struttura database
ogni utente ha il suo id, per creare l'accound deve inserire un'email (con cui si fara la 2FA), potrà accedere tramite email o userName e con la password, oppure tramite un codice per email (potrà scegliere se resettare la password oppure accedere tramite un codice da 6 cifre mandato per email)
ogni utente deve inserire il suo paese di residenza e le lingue che parla, la sua data di nascita (ci saranno dei libri con le restrizioni di età) e altro ...

i libri saranno salvati con id, il titolo (bisogna trovare un modo per salvare ogni titolo in base alla lingua), l'autore/autori, la casa editrice, rating, visualizzazioni, data di pubblicazione, tipologia (triller, comico ...) e altro ...
ci sarà una cartella dentro assets con dentro una directory per ogni libro, con all'interno la copertina e se ci sono le immagini all'interno dei libri, ed il contenuto dei libri, di cui sono ancora indeciso come salvare. stavo pensando di salvarli in semplici file di testo, salvandoli come se fossero un linguaggio di markup simile ad html, con <h1> <h2> <h3> per i titoli, i titoletti, le annotazioni... mentre il testo normale è lasciato cosi com'è. ci saranno cose come <pageEnd> per capire quando magari il testo dopo deve andare direttamente all'altra pagina. 
per i libri con le immagini all'interno, penso di metterle sempre come se fossero il linguaggio di markup: <img src"nome immagine" size ...> tanto le immagini saranno nella stessa cartella. Penso di salvare in file diversi il contenuto di lingue diverse

per fumetti li salvo sempre con gli stessi dati dei libri, ma ovviamente al posto del contenuto ci saranno le immagini numerate per pagina. La visualizzazione dei fumetti sarà diversa perchè vengono salvati in volumi delle stesse edizioni (the Batman, the Flash, the ultimate batman ...)

i manga verranno salvati in modo quasi uguale ai fumetti, cambia per esempio che i manga vengono divisi in capitoli, e generalmente non ci sono più edizioni dello stesso manga. stavo pensando, visto che la maggior parte dei manga è in bianco e nero, le immagini potrebbero essere salvate in bitmap per salvare spazio sul server (se la cosa ha senso). 

vorrei usare anche delle API per prendere i libri, fumetti, manga se esistono. 

i giornali pensavo di metterli come record della storia accaduta, salvandoli per titolo, editore, data uscita, argomenti, ... salvati penso in immagini o come si trovano 

ogni utente può salvare nelle sue liste (ci sarannò gia preimpostate le liste Preferiti e Da leggere) ogni oggetto. Verranno salvate per utente le letture, dove si è arrivati nella lettura ...
gli utenti possono commentare, mettere like (che salverà in automatico nei Preferiti) solo quando si ha letto una certa percentuale della lettura, e magari sottolineare parti importanti nei libri e nei giornali (una funzione un pò avanzata)

Diagramma ER aggiornato (coerente con la descrizione attuale)

users
- id (PK, int)
- email (string, UNIQUE)
- username (string, UNIQUE)
- password_hash (string, NULL se login solo OTP)
- full_name (string)
- role (enum: user, admin, author, editor)
- country_code (string)
- birth_date (date)
- avatar_dir (string)
- created_at (datetime)
- updated_at (datetime)

user_languages
- user_id (FK -> users.id)
- language_code (string)
- (user_id, language_code) (PK composita)

user_preferences
- id (PK, int)
- user_id (FK -> users.id, UNIQUE)
- theme (string)
- font_size (int)
- ui_language (string)
- updated_at (datetime)

user_sessions
- id (PK, int)
- user_id (FK -> users.id)
- session_token (string, UNIQUE)
- ip (string)
- user_agent (string)
- device_label (string, NULL)
- expires_at (datetime)
- created_at (datetime)

auth_challenges
- id (PK, int)
- user_id (FK -> users.id)
- purpose (enum: login, password_reset)
- channel (enum: email)
- challenge_token_hash (string, UNIQUE)
- expires_at (datetime)
- consumed_at (datetime, NULL)
- created_at (datetime)
# Nota privacy: il codice OTP non viene mai salvato nel DB.
# Verifica consigliata: OTP stateless con token firmato a scadenza breve.

publishers
- id (PK, int)
- name (string, UNIQUE)

catalog_items
- id (PK, int)
- type (enum: book, comic, manga, newspaper)
- fulfillment_type (enum: digital, physical, default digital)
- publisher_id (FK -> publishers.id, NULL)
- age_rating_min (int, default 0)
- release_date (date, NULL)
- price (decimal)
- currency (string)
- isbn (string, NULL)
- source (enum: internal, api)
- external_provider (string, NULL)
- external_id (string, NULL)
- avg_rating (decimal, NULL)
- views_count (int, default 0)
- created_at (datetime)
- updated_at (datetime)

catalog_item_translations
- id (PK, int)
- item_id (FK -> catalog_items.id)
- language_code (string)
- title (string)
- description (text, NULL)
- content_path (string, NULL) # file testo/markup per lingua
- content_format (enum: txt, html_like, markdown, image_sequence)
- (item_id, language_code) (UNIQUE)

series
- id (PK, int)
- name (string)
- type (enum: comic, manga)

series_entries
- id (PK, int)
- series_id (FK -> series.id)
- item_id (FK -> catalog_items.id)
- volume_no (decimal, NULL, ES 1, 2, 2.1, 2.5, 3)
- chapter_no (int, NULL)
- entry_order (int)
- (series_id, item_id) (UNIQUE)

item_media
- id (PK, int)
- item_id (FK -> catalog_items.id)
- language_code (string, NULL)
- media_type (enum: cover, page, image)
- storage_path (string)
- mime_type (string)
- sort_order (int)

authors
- id (PK, int)
- full_name (string)
- role (string, es scrittore)

item_authors
- item_id (FK -> catalog_items.id)
- author_id (FK -> authors.id)
- (item_id, author_id) (PK composita)

categories
- id (PK, int)
- name (string, UNIQUE)

item_categories
- item_id (FK -> catalog_items.id)
- category_id (FK -> categories.id)
- (item_id, category_id) (PK composita)

user_lists
- id (PK, int)
- user_id (FK -> users.id)
- name (string) # es. Preferiti, Da leggere
- is_system (bool)
- created_at (datetime)
- (user_id, name) (UNIQUE)

user_list_items
- id (PK, int)
- list_id (FK -> user_lists.id)
- item_id (FK -> catalog_items.id)
- added_at (datetime)
- (list_id, item_id) (UNIQUE)

library_items
- id (PK, int)
- user_id (FK -> users.id)
- item_id (FK -> catalog_items.id)
- source (enum: saved, purchased)
- added_at (datetime)
- (user_id, item_id, source) (UNIQUE)

orders
- id (PK, int)
- user_id (FK -> users.id)
- status (enum: pending, paid, failed, cancelled)
- total_amount (decimal)
- created_at (datetime)

order_items
- id (PK, int)
- order_id (FK -> orders.id)
- item_id (FK -> catalog_items.id)
- unit_price (decimal)
- quantity (int)

payments
- id (PK, int)
- order_id (FK -> orders.id)
- provider (enum: stripe, simulated)
- provider_tx_id (string, NULL)
- status (enum: pending, succeeded, failed)
- paid_at (datetime, NULL)

item_licenses
- item_id (PK + FK -> catalog_items.id)
- available_copies (int)
- updated_at (datetime)
# Nota: usata solo per item fisici o con copie limitate.

comments
- id (PK, int)

- item_id (FK -> catalog_items.id)

- created_at (datetime)
- updated_at (datetime)

item_likes
- user_id (FK -> users.id)
- item_id (FK -> catalog_items.id)
- created_at (datetime)
- (user_id, item_id) (PK composita)

highlights
- id (PK, int)
- user_id (FK -> users.id)
- item_id (FK -> catalog_items.id)
- language_code (string)
- locator_start (string)
- locator_end (string)

- note (text, NULL)
- created_at (datetime)

reading_progress
- id (PK, int)
- user_id (FK -> users.id)
- item_id (FK -> catalog_items.id)

- (user_id, item_id) (UNIQUE)
